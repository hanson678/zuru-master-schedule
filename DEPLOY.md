# ZURU 排期录入系统 v4.2 — 部署与使用指南

## 目录结构

```
排期系统/
├── 排期系统.exe         # 双击启动（打包版）
├── _internal/           # 运行依赖（勿删）
│   ├── templates/       # 网页模板
│   ├── static/          # 前端资源
│   └── data/            # 配置+直查表
└── uploads/             # PO上传临时目录（自动创建）
```

---

## 一、前提条件

| 条件 | 说明 |
|------|------|
| **Windows 系统** | 写入Excel 用 WPS COM，仅 Windows 可用 |
| **WPS Office 已安装** | 金山 WPS，需包含表格组件（Ket.Application） |
| **共享盘访问权限** | 需要能访问存放排期Excel文件的网络盘或本地文件夹 |

> **注意**：打包版（exe）无需安装 Python，直接双击运行。

---

## 二、安装与启动

### 打包版（推荐给其他电脑用户）

1. 将整个 `排期系统/` 文件夹复制到目标电脑
2. 双击 `排期系统.exe`
3. 浏览器访问：**http://localhost:5000**

### 源码版（开发用）

```bash
pip install -r requirements.txt
python app.py
```

---

## 三、首次使用配置

### 1. 设置排期文件路径

1. 打开系统首页 http://localhost:5000
2. 顶部有"**当前排期路径**"输入框
3. 点击"**切换**"→ 浏览到存放排期 Excel 文件的文件夹
4. 支持本地盘、网络共享盘（如 `Z:\各客排期\ZURU生产排期`）

路径会保存到 `data/config.json`，重启后自动记忆。

### 2. 确认排期文件状态

右侧面板"**排期文件状态**"会实时显示所有排期文件：
- 绿色 = 可用（可正常写入）
- 红色 = 被占用（有人正在编辑）
- 点击"刷新"按钮更新状态

---

## 四、使用方法

### 批量入单

1. 将 PDF 或 Excel PO 文件拖入上传区域（支持多个）
2. 系统自动解析 → 搜索排期 → 匹配货号
3. 结果分三类显示：
   - **橙色（排期被占用）**：货号存在但排期文件有人打开，可设定时间自动重试
   - **红色（货号不存在）**：排期中找不到该货号的参考行，需手动录入
   - **紫色（混装拒绝）**：混装箱拆分被拒绝，需手动处理
4. 确认无误后点击"**执行入单**"

### 定时重试入单

当有排期被占用时：
1. 在橙色提示区域选择重试时间
2. 点击"**设置定时入单**"
3. 系统后台每3分钟检查，到时间后自动执行

### 智能批处理

- 系统会**先录入未被占用的排期**，被占用的排到定时队列
- 入单时自动跳过被锁定的文件，不会因为一个文件被占用导致整批失败

### 搜索功能

顶部搜索框支持：PO号、货号、客户名、SKU 模糊搜索

## 五、常见问题

### Q: 文件状态显示"被占用"但实际没人打开？
A: 可能是 WPS 异常关闭留下的残留锁文件（~$开头）。系统已内置写入测试验证，正常情况不会误报。如仍有问题，手动删除排期目录下以 `~$` 开头的文件即可。

### Q: 提示"COM 错误"或"WPS 未找到"？
A: 确认 WPS Office 已安装，且版本包含 WPS 表格组件。

### Q: 端口 5000 被占用？
A: 修改 `data/config.json` 中的端口设置，或源码版修改 `app.py` 最后一行的 `port=5000`。

### Q: 一次上传多少个PDF最稳？
A: 建议 **10-15个PDF**（约50-80个货号），保证搜索不超时。

### Q: 换了排期文件路径后搜索不到货号？
A: 进入设置页面重新扫描直查表（item_schedule_map.json），或重启系统。

## 六、v4.2 更新内容

### 新增功能
1. **排期文件实时状态面板** — 显示所有排期文件的占用/可用状态，含编辑者信息
2. **三类错误分类** — 橙色（被占用）、红色（不存在）、紫色（混装拒绝），替代原来的统一红色警告
3. **定时重试入单** — 被占用的排期可设定时间自动重试，后台每3分钟检查
4. **智能批处理** — 先入未被占用的排期，被占用的自动排队
5. **锁检测写入验证** — 通过实际写入测试判断文件是否真锁，排除WPS异常关闭留下的残留假锁

### 优化改进
6. **ISM路径自动重映射** — 支持.xlsx0.xlsx被占用文件路径回退
7. **auto_find缓存清理** — 每次批量上传自动清理搜索缓存，避免旧结果干扰
8. **smart_diff路径锁检测** — 修改单也支持被占用文件检测（不仅限新单）

### v4.2.1 Bug修复（2026-03-10）
9. **修改单蓝色填充** — 修改单改动的单元格现在正确标记蓝色填充，方便识别改了哪些格
10. **新行不复制分配列** — 新录入行只复制到金额列为止，金额之后的分配列留空（用户手动处理）
11. **去重防护** — batch_process和smart_diff双重去重，防止同一货号重复写入多行
12. **策略3收紧** — smart_diff兜底匹配只在货号列(F/G/H)做，不在PO号列做，避免PO号与货号混淆导致乱入
13. **修改单取消检测** — 修改PO时，排期中同PO但新PDF中已不存在的货号自动生成取消操作

## 七、技术架构

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 框架 | Flask 3.1.3 | Python |
| Excel 写入 | WPS COM（Ket.Application） | Windows 专属 |
| Excel 读取 | openpyxl（read_only） | 只读搜索，不影响排期 |
| PDF 解析 | pdfplumber | ZURU PO PDF |
| 前端 | Bootstrap 5.3 + 原生 JS | 无框架依赖 |
| 数据存储 | JSON 文件 | 不依赖数据库 |
| 打包 | PyInstaller 6.19 | 单目录分发 |
